\chapter*{Resumen}
\thispagestyle{empty}  % Sin número en primera página del resumen
\addcontentsline{toc}{chapter}{Resumen}  % Añadir al índice

\textbf{Palabras clave:} automatización, testing, facturación, hosting, validación.

\vspace{0.5cm}

En este proyecto implementaremos una solución para validar la corrección de un sistema de facturación de servicios en la nube, que funcione de manera automatizada. Se desarrollará una API REST en Node.js que valide de manera cruzada los cargos que se hacen a través de APIs internas de la empresa, que garantice una facturación estable, fiable y basada estrictamente en el consumo de los clientes. El proyecto debe ser capaz de usar flujos de trabajo y servicios en la nube, y abordar temas
como la implementación, operación y monitoreo de aplicaciones REST de back-end.

\vspace{2cm}

% ===== ABSTRACT =====
\chapter*{Abstract}
\thispagestyle{empty}
\addcontentsline{toc}{chapter}{Abstract}

\textbf{Keywords:} automation, testing, billing, hosting, validation.

\vspace{0.5cm}

This project implements an automated quality assurance solution to validate Cloud billing accuracy. A Node.js REST API application regularly cross-checks product charges via internal APIs, ensuring stable and reliable consumption-based invoicing. The project provided hands-on experience in QA workflows and Cloud Services, as well as in the topic of implementing, operating and monitoring the back-end REST applications.

\vspace{2cm}
